Meet-Day Workflow
  1. In Splash Meet Manager: File → Export → Lenex → save as .lxf
  2. Copy the .lxf file to Pi #1 at ~/Scoreboard/meet/
  3. In the admin UI, go to Meet Setup and click Reload Names — swimmer and club names are now live on the scoreboard
  4. Optionally upload a Hytek .csv event schedule so event names appear in the header
  5. Open the scoreboard on the TV: http://10.10.10.10:5000/
  6. Start the CTS console — times appear automatically as heats run

Service Management (SSH into Pi #1)
sudo systemctl start scoreboard     # start the service
sudo systemctl stop scoreboard      # stop the service
sudo systemctl restart scoreboard   # restart after manual changes
journalctl -u scoreboard -f         # follow live logs
Troubleshooting
Scoreboard shows no times
Check the serial port in Meet Setup matches the adapter device (/dev/ttyUSB0 or /dev/ttyUSB1). Use the Debug tab to confirm packets are arriving from the console.
Swimmer names are not showing
Confirm the .lxf file is in ~/Scoreboard/meet/, then click Reload Names. Event and heat numbers must match between Splash and the CTS console.
The service won't start
Run journalctl -u scoreboard -f to see the error. Common causes: wrong serial port, missing Python dependencies (run uv sync), or a port conflict on 5000.
Settings page is unreachable
Confirm Pi #1 is powered and on the same network. The firewall only allows connections over eth0 — connect by ethernet cable, not WiFi.
NTP sync fails
NTP requires internet access. Connect Pi #1 to a WiFi network in the Network tab, then retry. Use the Time tab to set the clock manually if no internet is available at the pool.
Serial adapter not detected
Run ls /dev/ttyUSB* on Pi #1. Make sure the pi user is in the dialout group — run groups pi to check. If not, run sudo usermod -aG dialout pi and reboot.
